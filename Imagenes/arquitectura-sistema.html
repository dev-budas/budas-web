Ciclo de vida del lead
Flujo completo desde el formulario hasta el cierre
Formulario landing →
nuevo →
bot_enviado →
respondio →
calificado / →
no_calificado
calificado →
en_seguimiento →
visita_agendada →
captado / →
perdido
1. Lead llena formulario → POST /api/leads → se crea con status nuevo
2. Se envía mensaje WhatsApp de Silvina → status cambia a bot_enviado
3. Lead responde → POST /api/webhooks/whatsapp → status respondio
4. Silvina conversa (máx. 10 turnos) → llama a herramienta qualify_lead
5. Bot decide → status calificado o no_calificado → email a todo el equipo
6. Admin asigna agente → email al agente asignado
7. Agente hace seguimiento → puede agendar visita → status visita_agendada
8. Cierre → captado (éxito) o perdido

Notificaciones por email (Resend)
Lead calificado
Disparado desde el bot (automático) o cuando un agente/admin cambia el status manualmente. Se envía a todo el equipo. CTA: "Ver lead y asignar agente".
Lead no calificado
Igual que el anterior. Se envía a todo el equipo para que revisen la conversación de WhatsApp y decidan si reclasificar. CTA: "Revisar conversación".
Lead asignado a agente
Cuando el admin asigna un lead, o cuando un agente se auto-asigna. Se envía solo al agente asignado. CTA: "Ver lead".
Dominio de envío: no-reply@notificaciones.budasdelmediterraneo.com · Proveedor: Resend · DNS: Cloudflare
